Quick Start

启动顺序

先搭好工作区，再起网关，最后再起控制台。这样路径关系和配置都会更稳定。

1. 安装与初始化

在仓库根目录创建虚拟环境并安装当前源码。

python -m venv .venv
. .venv/bin/activate
pip install -e .

2. 准备 workspace

这些目录会保存会话、记忆、图谱和定时任务状态。

mkdir -p workspace/sessions
mkdir -p workspace/memory
mkdir -p workspace/cron

3. 启动 gateway

网关负责多通道接入、消息循环与代理执行。

mathclaw gateway \
  --workspace ./workspace \
  --channel wecom

4. 启动控制台

控制台单独运行在 console 目录，但工作区仍然指向仓库根目录下的 workspace。

cd console
MATHCLAW_CONSOLE_WORKSPACE=../workspace \
python serve.py
Workspace Flow

工作区里的数据如何流动

sessions

保存原始聊天记录，是所有后续记忆与图谱更新的第一层输入。

workspace/sessions jsonl

daily / weekly memory

对对话按天、按周进行总结，生成学习计划、重点纠错方向和优先复习知识点。

workspace/memory/daily_memory workspace/memory/weekly_memory

graphs

知识图谱与错题图谱的数据源，控制台图谱页面直接读取这里的 JSON。

workspace/memory/graphs knowledge_graph.json error_graph.json

cron

记录定时任务配置与运行状态，供心跳页查看最近一次执行、下次执行和执行结果。

workspace/cron/jobs.json heartbeat
Operator View

学生侧和管理侧各负责什么

学生工作区

解题工作台、学习计划和记忆图谱直接服务于学习本身：解题、复盘、补弱和形成后续训练计划。

管理工作区

运行状态、心跳、Skills、MCP 与模型配置则承担系统管理职责，让多通道与自动任务维持可控。
