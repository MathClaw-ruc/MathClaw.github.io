Channel Deployment ​

MathClaw is usually deployed as a shared runtime made of:

  • a gateway process for channels and agent execution
  • a console process for the web interface
  • one persistent workspace for memory, graphs, and heartbeat state

Recommended deployment pattern ​

  1. Keep the workspace on durable storage.
  2. Start gateway and console as separate long-running services.
  3. Enable only the channels actually required in production.
  4. Verify runtime and heartbeat pages after startup.

Common channel choices ​

  • WeCom for real teaching operations
  • QQ as a secondary user entry point
  • Feishu for enterprise collaboration scenarios
  • Web Console for operator access and demos
