通道部署 ​

生产环境部署

将 MathClaw 部署为稳定运行的服务

稳定部署的关键是让 gateway、console 和 workspace 分离但连通，确保外部通道、学习计划、图谱记忆和自动总结都能读写同一份数据。

GatewayConsoleWorkspaceChannels

Gateway 服务

负责通道接入、AI 解答执行，以及每轮会话后的记忆更新。

Console 服务

提供运行状态、图谱、计划、定时任务和技能面板的管理界面。

持久化 workspace

保存 sessions、memory、graphs 和定时任务状态，是所有页面的数据来源。

推荐部署模式 ​

Channels 企业微信、QQ、飞书或 Web 入口将学习任务送入系统。
Gateway 代理运行时负责执行，并将状态写入 workspace。
Workspace 会话、记忆、图谱和定时任务状态在此持续积累。
Console 管理者通过控制台查看系统状态、学习计划和图谱质量。

运维建议

  • 将 workspace 放在持久化存储上
  • gateway 和 console 作为两个长期运行服务分别启动
  • 只启用生产环境实际需要的通道

上线后优先检查

  • 运行状态：服务、provider、通道是否正常
  • 定时任务：日总结和周总结是否按计划执行
  • 记忆页：图谱数据是否正常更新

常见通道选择 ​

企业微信 适合真实教学运营和企业环境。
QQ / 飞书 适合作为学生入口或企业协作入口。
Web Console 建议始终保留，作为管理和演示入口。
