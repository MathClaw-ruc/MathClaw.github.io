Workspace & Memory ​

MathClaw keeps its learning state inside the workspace. The runtime depends on these directories staying connected.

Data layers ​

sessions ​

The raw conversation log. Every later memory update starts from here.

daily / weekly memory ​

Used to summarize study state and produce plan-oriented views such as review priorities and correction directions.

graphs ​

The JSON source for the knowledge graph and the error graph shown in the console.

cron ​

Stores scheduled task state for heartbeat monitoring.
