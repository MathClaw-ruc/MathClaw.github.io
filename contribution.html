Contribute ​

MathClaw is organized as a working product system rather than a single demo page. Contributions are most useful when they improve one of the following layers clearly:

  • Learning experience: solver workspace, study planning, graph readability, attachment output styles.
  • System operability: runtime status, heartbeat visibility, channel stability, configuration portability.
  • Memory quality: better graph extraction, more stable daily / weekly summaries, cleaner workspace data flow.

Recommended contribution areas ​

1. Product surfaces ​

  • Improve solver rendering for long mathematical answers.
  • Refine study planning information density.
  • Improve graph readability and node relationship presentation.
  • Make Skills, heartbeat, and runtime pages easier to operate.

2. Deployment and portability ​

  • Remove machine-specific assumptions from configs and startup scripts.
  • Improve startup paths, channel onboarding, and environment-variable handling.
  • Keep GitHub Pages and console-facing docs aligned with the actual repository structure.

3. Documentation ​

  • Keep README and docs aligned with the running product.
  • Prefer product-facing explanations over migration notes.
  • When updating screenshots, use the current MathClaw console rather than placeholder media.

Before opening a PR ​

  1. Verify the related page still renders correctly.
  2. Avoid including runtime history or transient workspace artifacts unless they are intentionally tracked.
  3. Keep brand naming consistent as MathClaw in user-facing surfaces.
