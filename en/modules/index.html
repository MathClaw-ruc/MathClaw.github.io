Modules

Break the product into clearer interface modules

MathClaw is more than a single chat surface. It combines student-facing learning pages with operator-facing control pages. This page shows what each module solves and how it appears in the live interface.

Module Showcase

Core pages

Solver Workspace

A single-dialog page for image/PDF upload, structured explanation, grading feedback, and follow-up problem solving.

  • Single conversation mode keeps the student focused
  • Structured tables and lists render directly in the workspace
  • Attachment upload and text input stay in one flow

Study Planning

A single learning panel for daily state, weekly targets, next-day actions, review priorities, and correction focus.

  • Shows students what to study first
  • Separates review topics from correction directions
  • Optimized for action rather than document reading

Knowledge Graph / Error Graph

Knowledge graphs track structure. Error graphs track repeated mistakes. Both share one visual grammar with different semantic colors.

  • Focus mode keeps the graph readable
  • Node labels and badges avoid overlap
  • Right-side cards summarize mastery and action hints

Runtime / Heartbeat / Skills

Runtime tracks health, heartbeat tracks scheduling stability, and Skills manage attachment-only output styles.

  • Designed for operators rather than students
  • Separates observability, automation, and teaching style control
  • Uses a shared visual language across operator pages
