Quick Start

Startup order

Build the workspace first, then start the gateway, and only then launch the console.

1. Install and initialize

Create a virtual environment and install the current source tree.

python -m venv .venv
. .venv/bin/activate
pip install -e .

2. Prepare workspace

These folders store sessions, memory, graphs, and scheduled task state.

mkdir -p workspace/sessions
mkdir -p workspace/memory
mkdir -p workspace/cron

3. Start gateway

The gateway handles channel connections, agent execution, and message loops.

mathclaw gateway \
  --workspace ./workspace \
  --channel wecom

4. Start console

The console runs from the console directory, but still points to the repository-level workspace.

cd console
MATHCLAW_CONSOLE_WORKSPACE=../workspace \
python serve.py
Workspace Flow

How data moves inside the workspace

sessions

Stores raw conversations and acts as the first input for all later memory and graph updates.

workspace/sessions jsonl

daily / weekly memory

Summaries of study state used to generate plans, priority review topics, and key correction directions.

workspace/memory/daily_memory workspace/memory/weekly_memory

graphs

Knowledge graph and error graph JSON files are rendered directly inside the console graph pages.

workspace/memory/graphs knowledge_graph.json error_graph.json

cron

Stores task configuration and task status for heartbeat pages and scheduled summary tasks.

workspace/cron/jobs.json heartbeat
Operator View

What belongs to student pages vs operator pages

Student Workspace

Solver, planning, and memory pages focus on learning itself: solving, reviewing, closing gaps, and planning the next move.

Operator Workspace

Runtime, heartbeat, Skills, MCP, and model pages keep deployment, automation, and style management observable.
