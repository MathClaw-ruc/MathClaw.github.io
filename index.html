Why MathClaw? ​

  • Math-first workflow — built around solving, review, memory, and long-term study progression rather than generic chat.
  • Structured memory runtime — sessions are converted into daily memory, weekly memory, knowledge graphs, and error graphs.
  • Student and operator surfaces — student pages focus on learning actions, while operator pages monitor runtime, heartbeat, and Skills.
  • Real deployment channels — the same agent can run in WeCom, QQ, Feishu, and the web console.
