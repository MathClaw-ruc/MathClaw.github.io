参与贡献 ​

MathClaw 不是单页演示，而是一套正在运行的产品系统。更有价值的贡献通常集中在下面三层：

  • 学习体验：解题工作台、学习计划、图谱可读性、附件输出风格。
  • 系统可运维性：运行状态、心跳、通道稳定性、配置可移植性。
  • 记忆质量：图谱抽取、日记忆 / 周记忆稳定性、工作区数据流一致性。

推荐贡献方向 ​

1. 产品页面 ​

  • 优化长篇数学回答的渲染效果。
  • 提高学习计划页面的信息密度和可读性。
  • 改进知识图谱、错题图谱的节点与关系展示。
  • 让 Skills、心跳和运行状态页更适合实际运维。

2. 部署与可移植性 ​

  • 去掉机器相关的硬编码路径。
  • 改善启动方式、通道接入和环境变量处理。
  • 让 README、文档与当前运行结构保持一致。

3. 文档 ​

  • 保持 README 和站点文档与真实功能一致。
  • 优先用产品文案，而不是迁移说明。
  • 更新截图时使用当前 MathClaw 界面。

提交前建议 ​

  1. 确认相关页面仍能正确渲染。
  2. 不要把临时运行产物误提交进仓库。
  3. 用户可见文案统一使用 MathClaw 品牌名。
