# 参与贡献

<div class="lead-panel">

## 高价值贡献方向

<div class="contrib-grid">
  <div class="contrib-card">
    <h3>学生体验</h3>
    <ul>
      <li>优化长篇数学答案的渲染效果</li>
      <li>提高学习计划的密度、节奏和可读性</li>
      <li>让图谱页在实际复盘场景中更易理解</li>
    </ul>
  </div>
  <div class="contrib-card">
    <h3>管理与运维</h3>
    <ul>
      <li>提升运行状态和定时任务页面的信息密度</li>
      <li>优化技能面板和配置页的操作体验</li>
      <li>减少对机器路径和部署环境的依赖</li>
    </ul>
  </div>
  <div class="contrib-card">
    <h3>记忆质量</h3>
    <ul>
      <li>改进图谱抽取和纠错信号的准确度</li>
      <li>提升日记忆和周记忆的稳定性与可读性</li>
      <li>保持 workspace 数据流在各页面之间一致</li>
    </ul>
  </div>
</div>

## 推荐贡献路径

<div class="steps-grid">
  <div class="step-card">
    <span class="flow-step">01</span>
    <h3>选一个具体模块入手</h3>
    <p>一次解决一个问题，比大范围重写更容易合并。</p>
  </div>
  <div class="step-card">
    <span class="flow-step">02</span>
    <h3>以当前产品为准</h3>
    <p>以现有控制台界面、路径结构和运行方式为事实来源，而非旧文档。</p>
  </div>
  <div class="step-card">
    <span class="flow-step">03</span>
    <h3>保持可移植性</h3>
    <p>使用相对路径和可迁移的配置，确保在不同机器上都能正常运行。</p>
  </div>
  <div class="step-card">
    <span class="flow-step">04</span>
    <h3>同步更新文档</h3>
    <p>功能变化时同步更新站点截图和文档，保持官网与真实系统一致。</p>
  </div>
</div>

## 提交前检查

<div class="issue-grid">
  <div class="issue-card">
    <h3>渲染检查</h3>
    <ul>
      <li>确认相关页面仍能正常渲染</li>
      <li>产品截图与真实界面保持一致</li>
    </ul>
  </div>
  <div class="issue-card">
    <h3>数据边界</h3>
    <ul>
      <li>不要误提交临时运行产物</li>
      <li>涉及 workspace 的变更要明确说明作用范围</li>
    </ul>
  </div>
  <div class="issue-card">
    <h3>品牌一致性</h3>
    <ul>
      <li>用户可见文案统一使用 MathClaw</li>
      <li>避免将内部迁移说明写进公开文档</li>
    </ul>
  </div>
</div>
