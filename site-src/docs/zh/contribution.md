# 参与贡献

<div class="lead-panel">
  <span class="kicker">贡献方式</span>
  <h1>围绕真实产品做贡献</h1>
  <p>高价值贡献通常不是孤立补丁，而是同时改善学习体验、管理体验和记忆运行质量。官网、运行结构和文档越一致，MathClaw 就越像一套真正可用的系统。</p>
</div>

## 高价值贡献方向

<div class="contrib-grid">
  <div class="contrib-card">
    <h3>学生体验</h3>
    <ul>
      <li>优化长篇数学答案的渲染效果。</li>
      <li>提高学习计划的密度、节奏和可读性。</li>
      <li>让图谱页在真实复盘场景里更容易理解。</li>
    </ul>
  </div>
  <div class="contrib-card">
    <h3>管理与运维</h3>
    <ul>
      <li>提升运行状态、心跳和通道页的判断效率。</li>
      <li>让附加输出和配置页更适合实际操作。</li>
      <li>减少机器相关路径和部署假设。</li>
    </ul>
  </div>
  <div class="contrib-card">
    <h3>记忆质量</h3>
    <ul>
      <li>改进图谱抽取和纠错信号的质量。</li>
      <li>提升日记忆和周记忆的稳定性与可解释性。</li>
      <li>保持 workspace 数据流在各页之间一致。</li>
    </ul>
  </div>
</div>

## 推荐贡献路径

<div class="steps-grid">
  <div class="step-card">
    <span class="flow-step">01</span>
    <h3>先选一个界面或一层运行时</h3>
    <p>一次解决一个连贯问题，通常比大范围重写更容易落地。</p>
  </div>
  <div class="step-card">
    <span class="flow-step">02</span>
    <h3>以当前产品为准</h3>
    <p>使用现有控制台截图、路径结构和运行方式作为事实来源，而不是依赖旧文档或占位逻辑。</p>
  </div>
  <div class="step-card">
    <span class="flow-step">03</span>
    <h3>保持可移植性</h3>
    <p>相对路径、可迁移的配置和稳定启动命令，与可见界面同样重要。</p>
  </div>
  <div class="step-card">
    <span class="flow-step">04</span>
    <h3>同步更新文档</h3>
    <p>功能变化时同步更新站点截图和文档，让官网与真实系统保持一致。</p>
  </div>
</div>

## 提交前检查

<div class="issue-grid">
  <div class="issue-card">
    <h3>渲染检查</h3>
    <ul>
      <li>确认相关页面仍能正常渲染。</li>
      <li>产品截图与真实界面保持一致。</li>
    </ul>
  </div>
  <div class="issue-card">
    <h3>数据边界</h3>
    <ul>
      <li>不要误提交临时运行产物。</li>
      <li>涉及 workspace 的变更要明确说明作用范围。</li>
    </ul>
  </div>
  <div class="issue-card">
    <h3>品牌一致性</h3>
    <ul>
      <li>用户可见文案统一使用 MathClaw。</li>
      <li>避免把内部迁移说明写进公开文档。</li>
    </ul>
  </div>
</div>
